Create these if they don't exist:


#### 3.1 API Client (`lib/api.ts`)
```typescript
// Axios instance with interceptors
import axios from 'axios';

const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL,
  headers: { 'Content-Type': 'application/json' },
});

// Request interceptor: attach JWT token
api.interceptors.request.use((config) => {
  const token = getToken(); // from cookie or localStorage
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Response interceptor: handle 401 (redirect to /login), format errors
api.interceptors.response.use(
  (res) => res,
  (error) => {
    if (error.response?.status === 401) {
      clearToken();
      window.location.href = '/login';
    }
    return Promise.reject(error.response?.data?.error || error);
  }
);

export default api;
```

## GLOBAL INTEGRATION RULES

1. **Location selector in TopBar** must write to `LocationContext`. Every data-fetching hook should read from it and include `?location={id}` in API calls. "All Locations" = no location filter (omit param).

2. **Loading states** — use skeleton/shimmer placeholders (not spinners) for table bodies and KPI cards.

3. **Empty states** — centered message + icon + CTA button (e.g., "No products yet. Add your first product.").

4. **Pagination** — use the `meta` object from API responses to render page controls. Default 20 items per page.

5. **Form validation** — validate client-side with Zod (same schemas or subset). Show server validation errors from API `error.fields` array under respective inputs.

6. **Optimistic updates** — not required. Just refetch/invalidate after mutations.

7. **File uploads** — use `POST /uploads` first, get back the URL, then include URL in create/update payload. OR send as multipart directly to the resource endpoint (categories, products, offers support this).

8. **Currency formatting** — always display as `₹{amount.toLocaleString('en-IN')}`. Never show $ or €.

9. **Date formatting** — display dates as `DD MMM YYYY` (e.g., "19 Jul 2026"). Use `date-fns` or `dayjs`.

10. **Status chips** — use the StatusChip component with the exact color mapping from the design system.
